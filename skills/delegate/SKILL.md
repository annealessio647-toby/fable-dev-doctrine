# 委托与保真（Delegate & Verify）

一句话：**报告是主张，不是证据；信任必须在每一层由独立验证重新挣得。**

## 角色分离（对人和对 AI 同样成立）

- **规划/判决**用最强认知：写 spec、拆任务、裁决审查发现、做终审。
  规划者不亲自淹在细节里——摘要与指针经手，全文留给执行方。
- **执行**一次一个任务，拿到的是**最小充分简报**：任务定位一句话 + 需求原文指针 +
  前序交付的接口契约 + 全局约束 + 已知环境坑 + 报告格式与状态协议
  （完成/带保留完成/缺上下文/阻塞）。**不给完整历史**——上下文是注意力预算，
  塞进无关历史等于稀释执行质量，还诱发越权发挥。
- **审查**由没参与实现的一方做，且必须**亲手重跑验证**（重跑测试、独立算法交叉
  验证、逐字节比对）——"实现者说测过了"永远只是主张。

## 保真五律

1. **义务传递**：任务 N 的审查发现若约束任务 N+1，必须写成硬性义务进 N+1 的
   简报，并要求 N+2 断言它。义务停在审查报告里=审查白做，缝隙静默穿过流水线。
2. **终审打集成缝**：逐任务审查只看切片，"每个切片都对但合起来错"是它的结构性
   盲区（生命周期洞、双端语义错位、组合竞态）。终审用最强认知专查跨单元交互
   ——本源战役中四个切片全绿后终审仍抓出一个组合僵死雷。
3. **发现不预判、不丢弃**：给审查者的是按风险排序的狩猎清单，不是"帮我确认没
   问题"；禁止在派发词里替审查者预判"这不算缺陷"。小问题全部记账，终审统一
   裁决（必修/后修/接受），关单逐条销账。
4. **计划自身也可被告**：执行方发现 spec 自相矛盾时，裁决权在计划作者——当场
   修 spec 并注明依据，不让下一个执行方再踩。计划与执行冲突永远显式裁决，
   不许执行方自行"猜作者意图"。
5. **状态协议要有"阻塞"出口**：执行方卡住必须能说"我缺 X"而不是硬编。收到
   阻塞时先补上下文，再升级认知强度，再拆小任务，最后才怀疑计划本身——
   永远不许"原样重试"。

## 防失忆（长战役必备）

- **台账**：每个任务完成即落盘一行（交付物标识 + 审查结论 + 遗留项）。
  记忆/上下文丢失后，台账 + 版本历史是唯一真相；恢复时**信磁盘不信回忆**，
  已完成的任务绝不重派（重派整段已完成序列是实测中最贵的失败模式）。
- 大块内容（简报、diff、报告）走文件交接，不粘贴进对话——粘贴的每个字都
  永久占据后续每一轮的注意力。

## 并行判据

有共享可变状态（同一批文件/同一台设备/同一份数据）的任务**严格串行**；
确无交集且接口已在 spec 冻结的才并行，并行的风险由后续一致性检查兜底。
拿不准时串行——并行省的时间赔不起一次合并事故。
