# 委托与保真（Delegate & Verify）

**分工声明**：本操典提供流水线、拆分规则、门与路由；每一步的领域内容
（章节怎么写、代码怎么审）由执行者自带——操典保证质量不在传递中失真。

一句话：**报告是主张，不是证据；信任必须在每一层由独立验证重新挣得。**

## 流水线（显式顺序，门没过不许进下一步）

```
① 写 spec ──门:spec 冻结──▶ ② 拆分 ──门:接口冻结──▶ ③ 逐单元循环
                                            ┌────────────────────────┐
                                            │ 简报→执行→独立审查→修复 │×N
                                            │ →义务传递→台账记一行    │
                                            └────────────────────────┘
──门:全部单元过审──▶ ④ 终审(专查集成缝) ──▶ ⑤ 统一裁决+逐条销账关单
```

## ① 写 spec（规划者产出，拆分的前置输入）

spec 至少含四块：**全局约束**（逐条可核对的硬要求）、**一致性规范**（术语表、
风格/格式/命名——凡多人产出后要拼在一起的，拼缝标准先冻结）、**单元间接口契约**
（交接点的内容与格式）、**验收判据**（每单元怎么算合格，量化到可判 yes/no）。
spec 没冻结前不许拆分——拆出来的边界会随 spec 漂移，全部返工。

## ② 拆分方法

1. **切分维度：按"可独立验收"切**——每个单元能单独判合格/不合格，不依赖
   别的单元先完成。能独立验收的边界就是好边界，反之切错了。
2. **依赖处理**：有依赖链的单元合并给同一执行方内部串行（依赖内化）；
   消不掉的跨人依赖点，上升为接口契约写进 spec 冻结。
3. **粒度准则：一个单元 = 一个审查门的工作量**——审查者一次能全量核完。
   大到审不完的拆小；小到审查开销超过工作本身的合并。
4. **拆完反验**：任意两个并行单元之间检查是否仍有共享可变状态（同一文件/
   设备/数据）——有就是拆错了，回去重切或改串行。拿不准时串行：并行省的
   时间赔不起一次合并事故。
5. **单元分派**：按执行方专长匹配单元；最弱的执行方拿依赖最少、最独立的
   单元（弱者的失误被隔离，不沿依赖链放大）。

## ③ 逐单元循环

- **角色分离**：规划/判决用可调用的最强认知（最强模型或最资深的人）；执行
  一次一单元；审查由没参与实现的一方做。人手不足时用交叉互审，但任何人
  不得审自己的产出。
- **最小充分简报**六要素：任务定位一句话 + 需求原文指针 + 前序接口契约 +
  全局约束 + 已知坑 + 报告格式与状态协议（完成/带保留完成/缺上下文/阻塞）。
  **不给完整历史**——上下文是注意力预算，塞无关历史=稀释质量+诱发越权发挥。
- **独立审查**：给审查者按风险排序的狩猎清单（点名缺陷类别），禁止预判
  "这条不算问题"。审查者必须**亲手重做验证**——把执行方的每条关键声明变成
  自己动手的核对动作（声明"测过了"→亲自重跑；声明"符合规范"→逐条比对
  规范原文；声明"数据有出处"→抽样回溯源头）。
- **义务传递**：单元 N 的审查发现若约束单元 N+1，写成硬性义务进 N+1 简报，
  并在 N+2 的验收清单里显式核查落实。义务停在审查报告里=审查白做。
- **阻塞路由**（执行方报"缺 X"、或同一单元修复 2 轮仍不过审时，按序升级，
  永远不许原样重试）：
  补上下文重派同一执行方 → 换更强的执行方 → 把单元拆小 → 怀疑 spec 本身
  （回到规划者裁决）。执行方凭空编造着继续做是最坏结局，状态协议就是为
  堵死它存在的。
- **spec 争议**：执行中发现 spec 自相矛盾，裁决权在 spec 作者——当场改 spec
  注明依据并同步所有未完成单元，不许执行方猜作者意图。
- **台账**：每单元过审即落盘一行（单元号+交付物标识+审查结论+遗留项）。
  记忆丢失后台账+版本历史是唯一真相，已完成单元绝不重派。大块内容
  （简报/diff/报告）走文件交接不粘贴，粘贴的每个字永久占据后续注意力。

## ④ 终审（最强认知，专查集成缝）

逐单元审查只看切片，"每个切片都对但合起来错"是它的结构性盲区。终审只查
跨单元交互：拼缝一致性（术语/口径/风格断层）、交接点双方实现是否同义、
组合时序/组合条件（实例：四个任务各自测试全绿，终审才发现两个"各自正确"
的等待条件组合成相互卡死）、整体对 spec 全局约束的覆盖有无漏项。

## ⑤ 裁决与关单

终审把台账上全部遗留发现逐条裁决：必修/后修/接受。必修清零、销账记录
落盘后才算关单——没人读的遗留清单等于静默丢弃。
