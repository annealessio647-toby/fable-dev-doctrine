# 行动棘轮（Ratchet Discipline）

**分工声明**：本操典提供三段流程、段间门、判据制定框架与失败路由；每步的
领域参数（具体测什么、调多少）由执行者自带——操典保证只向前不滑落。

一句话：**每一步只向前不滑落——可回滚保下界，判据锁上界，沉淀把这一步永久焊死。**

## 三段与两道门

```
【之前】准备 ──准入门──▶ 【之中】小步循环 ──关单门──▶ 【之后】沉淀
准入门（缺一不许动手）：          关单门（缺一不算完）：
□ 判据已写进工单                  □ 哨兵指标已挂上
□ 基线已测出                      □ 双份沉淀已落盘
□ 回滚路径已备好且演练可行        □ 临时状态已复原并回读核对
□ 不可逆动作已单独设门            □ 判决书已写
```

## 之前：准备

1. **判据制定三问**（回答不出=还没准备好）：
   - **测什么**：哪个可直接测量的量最接近目标本身？警惕代理指标（"报警没响"
     不等于"没问题"）。写死测量口径：在哪测、谁测、何时测。
   - **多少算变**：先测**基线**——不动任何东西时该量的正常波动范围。阈值必须
     设在基线波动之外，效应小于基线噪声的"改善"不可判，等于没测。
   - **测多少**：多大样本/多长时间足以把"真变化"和"碰巧"分开？按基线波动
     大小定，波动越大要的样本越多；拍脑袋的 N 不算数。
   - **绿地新增**（做的是全新功能，没有旧基线可测）：基线=无此功能的现状，
     三问退化为验收双证据——功能确实被触发过 + 行为符合预期，仍缺一不可；
     "复现"一问对应跳过，其余门照走。
2. **判据要防空转通过**（被测机制从头到尾没被挑战过，绿灯却亮了）：既要证据
   证明故障/异常真的被触发过，也要证据证明机制把它化解了，缺一不可。
   顺带验证改动确实生效（改完回读，别测了半天测的是没生效的旧配置）。
3. **动作按可逆性分级**：秒级可撤销的=小步快跑不请示；恢复需要代价的=先备份
   快照再动；不可逆/对外发布/改共享资源的=单独设门（确认、备份、影响面清单：
   列出谁会被波及、怎么验证没伤到他们）。批量授权带过期条件。
4. **修复先复现**：改进前先证明现状可稳定观测（连续几个窗口的基线一致）——
   否则改完无法区分"真好了"和"碰巧没发作"。

## 之中：小步循环与失败路由

- **一次只改一个变量**，改完立即按判据验证，通过才进下一步；验证不过绝不
  叠加下一改动（叠加=把两个未知数搅成一锅）。多个变量待调时，按
  "预期效应大、失败代价小"者优先排队。
- **试验产出物隔离**：验证窗口内产生的一切产出物（数据、构建物、实物件）
  打隔离标识，判定通过前不得流入正常交付。
- **失败路由**（验证不过时按序走，禁止跳步、禁止原地蛮干）：
  ```
  ① 回滚到已知好状态 + 回读确认（先恢复棘轮下界，再想别的）
  ② 审判据本身：测量口径对吗？有混杂变量吗（同时换了料/换了班/换了环境）？
     **验证工具/测试环境自身也是嫌疑**——先证明观测通道可信再给产品定罪
     （测试机缺字体、模拟器与真机差异、探针自身故障；同 evidence 操典
     "监控显示正常先审监控"）
  ③ 减小步长重试同一变量
  ④ 放弃该变量，换下一个
  ⑤ 同一目标沿此路由走到底 3 次仍不过 ⇒ 停手，怀疑方案/架构本身，
     升级请示——第 4 次蛮干是最贵的选项
  ```
- **如实报告是硬约束不是美德**：失败写失败、跳过写跳过、回退数据标注[回退]。
  每一个虚假的"已解决"都是后续推理的沙地地基。工具/测试的职责是**表征与
  暴露**，绝不为让结果好看而调参、旁路、掩盖。
- **状态复原带回读**：改过的环境用完必须复原，且逐项打印"期望值→实测值"
  核对。顺序：先把易失的临时值写回原值，最后才执行会持久化/固化的提交类
  动作——顺序反了，持久化会把没复原的临时值永久焊死。
- **会话卫生**：接手先清账（排空积压、核对账面状态=实际状态），交还先结账
  （停掉自己启动的一切，异常退出路径也要停——兜底写在必然执行的清理路径里，
  如 finally/trap/交接检查单）。

## 之后：沉淀（复利部分——纪律的意义所在）

1. **哨兵指标**：每个改动交付时附带一个"此病复发立刻可见"的日常观测量
   （新基线控制线、"日志再出现这行=新问题"），写进日常巡检的文档。
   没有哨兵的修复会在半年后以变异形态复发且无人认得它。
2. **双份沉淀**：可复用的**方法**放按需查阅的工具文档（skill/SOP），恒久的
   **事实/地雷**放每次必然载入的备忘（CLAUDE.md/memory/交接板首页）。
   只存一处的知识活不过一次失忆——两处同步，缺一处等于没记。
3. **被纠正即沉淀**：被指出错误时先找错的默认假设再更新做法；同类错误
   第二次出现就固化成明文规则——两次是巧合，三次是流程缺陷。
4. **判决书**：根因 + 排除证据 + 前后对照数字（基线→新值、样本量）+
   新增铁律。写给三个月后失忆的自己和从未参战的下一个人。
