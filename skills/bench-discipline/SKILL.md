# 台架剖面（Bench Profile）

硬件的特殊之处：错误会伤人、烧板、丢不可再采的数据。以下是**不可妥协项**，
通用流程（判据先行、状态复原、哨兵指标、双份沉淀）见 `ratchet-discipline`，
排障方法见 `evidence-loop`。

## 转动机构

- **停机双验**：会话结束显式停速+断使能后，再用**物理量证明静止**——间隔 1s
  的位置双采样差≈0。不信任脚本 finally 的自觉，finally 可能根本没执行。
- 每个会碰电机的脚本，finally 必须带停机兜底（停速→断使能→停流），
  异常路径也要停。
- 长时间连续运转选保守转速——采集质量通常与转速无关，风险却成正比。

## 固件烧录

- 永远弹窗确认（哪个镜像、哪块板）；烧前验芯片 ID，不对拒烧；探针必须唯一
  （检测到多个编程器→停，不猜）。
- 烧完验证激活：新固件要有可观测的版本指纹，"烧了"≠"跑的是新的"。

## 共享固件的零触碰证明

- 声明"没动高危区（采样 ISR 链等）"必须**机器可验**：目标文件 md5 逐位一致
  （最强）或反汇编逐条指令 diff（次强）。"我没改那个文件"不是证明——
  头文件、对齐、字面池的连锁反应会背叛你。
- 每次重编译后，调试用的内存地址一律从新 map 文件重取，旧地址读到的是幻觉。

## 实物接线

- **逐根对照接线文档验收**，特别是地线/屏蔽/公共端——信号线会"看起来能用"，
  缺失的公共地是定时炸弹，症状伪装成"偶发 EMI"可以骗你几个星期。
- 文档与实物任何一边改动，同一次提交里同步另一边；"文档是对的、实物歪了"
  与"实物是对的、文档旧了"同样致命。

## 数据链路

- 存储转发型中继（桥、缓冲、队列）会把上一场会话的积压倒给下一场——
  会话开始主动排空到只剩心跳，单次静默窗判据会被积压中的暂停骗过。
- 微秒级介质上的时序侥幸（"50ms 排空肯定够"）换到毫秒级介质必然爆炸——
  跨介质移植时把所有"依赖时序运气"的假设列出来逐个审。
