# 证据收敛环（Evidence Loop）

一句话：**叙事是嫌疑人，证据是法官；每一轮行动的目的是让假设空间单调收缩。**

## 循环本体（对任何问题通用）

```
现象 → 全空间列举 → 选最便宜的判别器 → 执行 → 剪枝并留档 → 收窄 → 重复
                                                    ↑ 空间不许无新证据地变大
```

1. **全空间列举**：把现象经过的每一层/每个环节列成嫌疑清单，给每个嫌疑写出
   它"作案"时的**专属指纹**（会留下什么可观测量、什么相关性、什么量纲特征）。
   写不出指纹的嫌疑=你还不理解它，先补理解再列。
2. **判别器（discriminator）**：一个好实验让两个候选给出**相反**预期，一次剪一半；
   两个候选预期相同的实验是零信息，不许跑。判别器按成本排序：现成开关 > 换个输入 >
   加观测 > 改系统。问自己："哪个单一观测能让这两个解释分道扬镳？"
3. **剪枝要留档**：洗白记录与定罪同等重要——写下"层 X 无罪，证据 Y"，
   同一嫌疑不许无新证据地二次受审。这让搜索单调收敛而不是绕圈。
4. **定罪标准**：到最底层证据为止（字节、原始记录、现场状态值），
   "高度吻合的解释"不是定罪。修复后必须让原复现实验从必现变零现，且全量回归无损。

## 注意力法则

- **反直觉数据是门，不是噪声。** 与模型矛盾的观测是全场最贵的信息：
  "负载越重反而越快"直接指认空闲路径；"误差恒等于工作点的固定百分比"指认口径
  问题而非物理噪声（真噪声不会恰好正比于设定值）。抓矛盾，别抓佐证。
- **"重试就好了"是读取端/消费端缺陷的经典指纹**——东西一直在，是取的人不认识它。
  凡"第二次总能成功"，先审取的一方。
- **"监控显示正常"先审监控。** 计数器为零要先证明计数器在当前条件下可信
  （事件队列溢出、日志截断、聚合显示只露头部——观测通道自己也是嫌疑层）。
- **偶发 ≠ 随机。** 先找全部发作实例的公共因子（时间点、前置动作、状态），
  公共因子就是触发条件，触发条件就是复现器。

## 真相的两级制

- **凡有唯一源的事实（代码、文档、测量、原始记录）一律现查，禁止凭记忆/惯性引用。**
  记忆只用来定位唯一源在哪。被复述过 N 手的"事实"自动降级为待验传闻。
- **测量先于争论。** 当理论分支超过两个还在增殖，停止推理，去造一个测量。
  一次测量的成本 < 三个错误理论的成本。争不出来的问题基本都是缺一个数。

## 对抗自己的叙事

- 每个漂亮解释诞生时，立刻写出它的**可证伪预言**；下一个实验优先尝试杀死它
  而不是喂养它。解释活过刺杀才配升级为结论。
- **被质疑时先找自己推理的洞，再辩护。** 对方一句"就一台设备连着，你拥挤什么"
  可能正好戳中你从未验证的默认假设。承认"这是推断不是测量"不丢分，
  建立在推断上的三层大厦塌了才丢分。
- **环境优先原则**：昨天好、今天坏、期间零改动 ⇒ 先问"物理/环境/别人动过什么"，
  再查自己的代码。持有系统外状态的人（用户/运维/现场）是你最便宜的传感器，
  早问、具体地问。

## 交付物

结案必须产出**判决书**：根因 + 每层洗白证据 + 修复与回归数字 + **哨兵指标**
（一个"此病复发立刻报警"的日常可观测量）+ 本案新增的铁律。写给三个月后失忆的自己。
